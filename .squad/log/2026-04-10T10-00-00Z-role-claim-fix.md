## Key Decisions

1. **JWT Claims Specification:** Use short form `"role"` (not `ClaimTypes.Role` URI) in AuthController when adding role claims to JWT tokens
2. **Future Claims:** Any future claim additions must verify the JWT payload key that will actually be written
3. **MAUI AuthService:** No changes required; already expects `"role"` key in raw JSON decode

## Technical Details

**Root Cause:** When calling `JwtSecurityTokenHandler.WriteToken()` on a pre-built `JwtSecurityToken`, the `OutboundClaimTypeMap` is NOT applied. This caused the full URI `http://schemas.microsoft.com/ws/2008/06/identity/claims/role` to be written instead of the short form `"role"`.

**Impact:** MAUI's raw JSON JWT payload decode searches for `"role"` key. The URI key was never found, leaving `CurrentUser.Roles` empty. Crew app's role gate denied all logins.

**Fix Scope:** Single line change; no MAUI code changes; no schema changes.
