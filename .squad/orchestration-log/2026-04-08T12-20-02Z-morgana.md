## Summary

Morgana delivered the frontend UI for event report downloads. Implemented ReportDownloadButton component with inline section picker, integrated with the backend report endpoint. Handles role-based visibility, auth, and blob download pattern.

## Deliverables

### Components
- **ReportDownloadButton.tsx** — Main component with conditional rendering based on event status and user role
  - Section picker: inline collapsible panel (not modal)
  - Checkboxes for Registrations, CheckIns, Equipment, Tournaments (all default checked)
  - Auth: raw fetch with Bearer token from localStorage
  - Error handling: inline error message with clear-on-retry behavior

### API Integration
- **events.ts** — Added `downloadEventReport` function
  - Uses raw `fetch` (not `apiFetch`) for proper blob handling
  - Respects `Content-Disposition` header for filename
  - Returns typed `EventReportResponse` with file metadata

### Visibility Rules
- Component renders only when:
  - Event status = `Closed`
  - User role includes `Admin` or `Organizer`
- Fast feedback: no API call for unauthorized users

## Technical Decisions

1. **Inline panel over modal** — keeps action in-context on event detail page
2. **Raw fetch for blobs** — `apiFetch` would override headers and break streaming
3. **Sections=All shorthand** — cleaner URL than comma-separated section names when all selected
4. **Role check in component** — provides immediate UX feedback before backend validation

## Conventions Established

- `frontend/src/components/` directory created for shared reusable components
- PDF download pattern: `fetch → blob → createObjectURL → anchor click → revokeObjectURL`
