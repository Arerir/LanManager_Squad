## Summary

Circe implemented report download and share functionality in the LanManager.Maui.Crew app. Added section picker on AttendancePage with File.SaveAsync and Share.RequestAsync integration. Mirrors frontend section picker UX for consistency.

## Deliverables

### MAUI Components
- **AttendancePage.xaml** — Added section picker below attendance list
  - Section toggles: Registrations, CheckIns, Equipment, Tournaments (all default checked)
  - Download button: triggers `DownloadReportCommand`
  - Share button: triggers `ShareReportCommand` (Share.RequestAsync)

### ViewModel Integration
- **AttendanceViewModel.cs** — Added report download/share commands
  - Role gate: only enabled when user has Admin or Organizer role
  - Section flag computation: converts toggles to ReportSections enum
  - API call: `httpClient.GetAsync(reportUrl)` with Bearer token
  - File handling: `File.SaveAsync` on mobile, `Share.RequestAsync` for native sharing

### Platform Support
- iOS: Share.RequestAsync → native iOS share sheet
- Android: Share.RequestAsync → native Android share sheet + system file associations
- Windows (Desktop): File.SaveAsync → standard file picker dialog

## Technical Decisions

1. **Mirrors frontend picker** — consistent UX across web and MAUI
2. **Section flags as toggles** — users select exactly what they need
3. **Share.RequestAsync over direct download** — leverages native OS features
4. **Role gate in ViewModel** — command disabled for non-authorized users

## Code Patterns

- `HttpClient` with `GetAsync` for blob response
- `File.SaveAsync` with default filename pattern: `{EventName}-report.pdf`
- `CommunityToolkit.Mvvm` async command support for async/await
